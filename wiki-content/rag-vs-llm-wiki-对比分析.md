# RAG vs LLM Wiki

## RAG（检索增强生成）

RAG 是目前最主流的知识库方案，其核心思路是：在用户提问时，从大量文档中检索出最相关的片段，然后将这些片段作为上下文输入给 LLM 生成回答。

### 优点
- 实现简单，不需要预处理文档
- 可处理大规模文档库
- 实时性好，新增文档立即可用

### 缺点
- 检索质量依赖向量模型
- 可能检索到不相关或过时的内容
- 知识之间缺乏组织结构

## LLM Wiki

采用编译思路：先用 LLM 将文档处理成结构化 Wiki 页面，查询时直接从组织好的页面获取信息。

### 优点
- 知识经过提炼，质量更高
- 页面间有链接关系，便于导航
- 支持知识健康检查（Lint）
- 更接近人类整理知识的方式

## 对比总结

| 特性 | RAG | LLM Wiki |
|------|-----|----------|
| 知识组织 | 扁平文档库 | 结构化 Wiki |
| 检索方式 | 向量相似度 | 语义匹配 |
| 知识质量 | 依赖原文 | LLM 编译提炼 |
| 维护成本 | 低 | 中等 |
| 适用场景 | 大规模文档查询 | 个人知识管理 |
