# 知识管理系统设计

个人知识管理（PKM）系统的设计理念和方法论，以及如何用技术实现。

## 核心理念

> [!note] 知识管理的本质
> 知识管理不是"存储信息"，而是"建立连接"。好的知识系统帮助你在需要的时候找到需要的知识，并发现知识之间的关联。

## PKM 方法论

### Zettelkasten（卡片笔记法）

核心原则：
1. **原子性**：每个笔记只包含一个想法
2. **自主性**：每个笔记可以独立理解
3. **链接性**：通过双向链接建立网络

```
笔记 A ──引用──→ 笔记 B
  ↑                  │
  └────反向链接──────┘
```

### PARA 方法

| 分类 | 说明 | 示例 |
|------|------|------|
| Projects | 有明确目标的项目 | "重构 Wiki 系统" |
| Areas | 持续关注的领域 | "前端开发"、"AI 研究" |
| Resources | 参考资料和素材 | 技术文档、教程链接 |
| Archives | 已完成/不活跃的内容 | 上一个项目的资料 |

### Creators 方法（Tiago Forte）

1. **Capture**：收集所有有价值的信息
2. **Organize**：按行动导向分类
3. **Distill**：提炼精华，渐进式总结
4. **Express**：输出和分享

## 评估标准

好的知识管理系统应该满足：

| 标准 | 说明 |
|------|------|
| 低摩擦 | 记录想法的速度越快越好 |
| 可搜索 | 能快速找到已有知识 |
| 可连接 | 知识之间有关联 |
| 可回顾 | 有机制促进定期回顾 |
| 可导出 | 数据不被锁定在系统中 |
| 可备份 | 有可靠的备份和恢复机制 |
