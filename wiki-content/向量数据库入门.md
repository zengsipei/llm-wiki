# 向量数据库入门

向量数据库是存储和检索高维向量的专用数据库，是 RAG 系统的核心组件。

## 为什么需要向量数据库

> [!note] 向量检索的本质
> 传统数据库按精确匹配查询（WHERE name = 'xxx'），向量数据库按语义相似度查询（找到"意思最接近"的内容）。

| 场景 | 传统数据库 | 向量数据库 |
|------|-----------|-----------|
| 语义搜索 | ❌ | ✅ |
| 相似推荐 | ❌ | ✅ |
| 去重检测 | 困难 | ✅ |
| RAG 系统 | ❌ | ✅ |

## 主流向量数据库对比

| 数据库 | 类型 | 语言 | 特点 |
|--------|------|------|------|
| Pinecone | 云服务 | - | 全托管，零运维 |
| Weaviate | 开源/云 | Go | 内置向量化模块 |
| Milvus | 开源 | Go/C++ | 高性能，分布式 |
| Qdrant | 开源 | Rust | 轻量，性能优秀 |
| Chroma | 开源 | Python | 简单易用，适合原型 |
| pgvector | 扩展 | C | PostgreSQL 扩展 |

## 核心概念

### 向量（Embedding）

将文本转换为高维数值数组（通常是 768-3072 维）：

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('bge-large-zh-v1.5')
vector = model.encode("如何学习机器学习？")
# 输出: [0.023, -0.041, 0.078, ...] (1024 维)
```

### 相似度计算

| 方法 | 公式 | 范围 |
|------|------|------|
| 余弦相似度 | cos(A, B) | [-1, 1] |
| 欧氏距离 | ||A - B|| | [0, +∞) |
| 点积 | A · B | (-∞, +∞) |

> [!tip] 推荐使用余弦相似度
> 余弦相似度对向量长度不敏感，只关注方向，适合文本检索场景。

## Chroma 快速上手

```python
import chromadb

client = chromadb.PersistentClient(path="./chroma_db")
collection = client.get_or_create_collection("wiki")

# 添加文档
collection.add(
    documents=["机器学习是人工智能的子领域", "深度学习使用神经网络"],
    metadatas=[{"source": "ai-wiki"}, {"source": "ai-wiki"}],
    ids=["doc1", "doc2"]
)

# 查询
results = collection.query(
    query_texts=["什么是神经网络？"],
    n_results=3
)
```

## 优化策略

1. **索引选择**：
   - IVF Flat：平衡精度和速度（推荐）
   - HNSW：高精度，内存消耗大
   - Flat：暴力搜索，小数据集适用

2. **过滤优化**：结合元数据过滤缩小搜索范围

3. **批量操作**：插入和查询都应使用批量接口

4. **定期维护**：清理过期数据，重建索引
