# Claude 使用指南

Claude 是 Anthropic 公司开发的大语言模型，以其安全性和长上下文能力著称。本页面汇总了 Claude 的核心使用技巧和最佳实践。

## 核心能力

Claude 的优势领域包括：长文档理解和分析、复杂推理任务、代码编写与调试、创意写作和结构化内容生成。其上下文窗口可达 200K tokens，适合处理超长文档。

## Prompt Engineering 技巧

### 1. 系统提示（System Prompt）

系统提示是定义 Claude 行为的最重要工具。好的系统提示应该：

- 明确角色定位：告诉 Claude 它是谁，应该用什么风格回应
- 设定行为边界：哪些可以做，哪些不应该做
- 提供输出格式：如果需要特定格式，直接在系统提示中说明
- 给出示例：Few-shot 示例比纯规则说明更有效

```
你是一个专业的技术文档审阅者。你的任务是：
1. 检查文档中的技术准确性
2. 标注模糊或矛盾的表述
3. 提出改进建议

输出格式：对每个问题，使用以下结构：
- 📍 位置：段落编号
- ⚠️ 问题：简要描述
- 💡 建议：改进方案
```

### 2. 结构化输出

Claude 在生成结构化内容方面表现出色。使用以下技巧获得更好的结构化输出：

- 使用 Markdown 标题层级组织内容
- 用编号列表确保步骤清晰
- 在提示中明确要求使用表格或特定格式
- 使用分隔线（---）区分不同部分

### 3. 长文档处理

Claude 的 200K 上下文窗口意味着你可以一次性处理非常长的文档：

- 直接粘贴完整文档，无需分段
- 要求 Claude 先总结，再深入特定部分
- 使用 "请引用原文" 来确保回答有据可查
- 对比分析时，可以将多份文档一次性输入

## 最佳实践

- **迭代优化**：第一次的 prompt 很少完美，持续优化
- **温度设置**：需要精确答案时用低温度（0-0.3），创意任务用高温度（0.7-1.0）
- **分步思考**：复杂任务让 Claude "一步步思考"
- **角色扮演**：为特定任务设定专业角色可以显著提升输出质量
- **反馈循环**：告诉 Claude 哪些回答好、哪些不好，让它自我修正

## 常见问题

**Q: Claude 会编造信息吗？**
A: 是的，Claude 也会产生幻觉（hallucination）。关键缓解方法：要求引用来源、使用 "如果你不确定，请说不知道"、交叉验证关键事实。

**Q: 如何让 Claude 的回答更简洁？**
A: 在系统提示中明确 "回答简洁，不超过 200 字" 或 "直接给出答案，不要解释过程"。

**Q: Claude 适合写代码吗？**
A: 非常适合。Claude 在代码生成、调试、重构和代码解释方面表现优秀。特别擅长阅读和理解大型代码库。
