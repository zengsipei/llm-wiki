## 会话 4: TOC 目录系统 (深度调试)

### Bug: 点击跳转后标题被吸顶栏遮挡
> 用户 (附截图): "功能都实现了。但是点击目录跳转还是有点显示问题，可能滚动后开始顶部没有预留间距，选中跳转的标题被隐藏在吸顶的操作栏下面看不到"

**修复历程**:
1. 添加 `scroll-margin-top: 80px` → 在嵌套容器中不生效
2. 改为 `mainEl.scrollTo({ top: el.offsetTop - 72 })` → offsetTop 不准确
3. 最终用 `getContentOffset()` 公式 + `mainEl.scrollTo()`

### Bug: 高亮不跟随阅读位置
> "当前依旧是不准确，有没有更简单更现代的友好解决方式呢。比如我看 zread 产品的文档详情，总是以视窗内最上面的标题来作为目录定位。类似这样的实现？"

**关键洞察**: 算法方向搞反了！
- 旧: 找最后一个 `top ≤ 阅读线` → 高亮已滚过去的标题
- 新: 找第一个 `top ≥ 阅读线` → 高亮视窗内最上面的标题

### Bug: 高亮仍然不准确 (3 个问题)
> "1. 高亮基本对了，虽然还是从吸顶下算的，但是已经没有偏那么多了 2. 点击跳转依旧不对，就不能根据高亮那样同步一下么 3. TOC 扫描到了代码块内的注释作为标题，这是 bug"

**根因分析**: `offsetTop` 的 `offsetParent` 链不经过 `<main>`（因为 `<main>` 没有 `position: relative`）

### 最终方案
> 用户确认: "完美！功能性上都没问题了。"

### Bug: Hover 交互不顺滑
> "TOC hover 窗口跟进度条之间的互动，从进度条移动到浮窗上大部分会失败，因为离开进度条hover，浮窗隐藏了，但是我明明鼠标已经在浮窗上了"

**首次修复**: 300ms 延迟定时器
> 用户反馈: "这种需要等加载一下么。感觉开始的几次还是不顺滑。有没有可能用伪类将面板与灯条搭边呢，这种可行么"

**最终方案** (用户提出思路):
